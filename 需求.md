网易云音乐自动打卡项目 - AI生成提示词整理

一、项目核心功能

通过访问接口 "/api/sign?key=123" 实现一键签到，结合 cron-job.org 实现自动定时打卡。  

二、参数与环境配置

1. 签名密钥（key）：通过环境变量设置，需在 .env 文件中配置以下变量：  
   • NETEASE_COOKIE（字符串类型）：存储网易云音乐账号的 Cookie（用于身份验证）。  

   • AUTH_KEY（字符串类型）：设置“一键签到”的密钥（需与访问接口的 key 参数对应）。  

   • PLAYLIST_ID (数字) 听歌打卡歌单id

   • PORT（数字类型）：项目运行端口，默认 3000。  

三、签到逻辑实现

签到功能通过 NeteaseCloudMusicApiEnhanced 实现, 官网地址：neteasecloudmusicapienhanced.js.org，需安装并正确引入相关模块。

模块调用示例：

```javascript
const {
  login_cellphone,
  user_cloud,
} = require('@neteasecloudmusicapienhanced/api')
async function main() {
  try {
    const result = await login_cellphone({
      phone: '手机号',
      password: '密码',
    })
    console.log(result)
    const result2 = await user_cloud({
      cookie: result.body.cookie, // 凭证
    })
    console.log(result2.body)
  } catch (error) {
    console.log(error)
  }
}
main()
```

其中需要实现的模块需要

听歌打卡
说明 : 调用此接口 , 传入音乐 id, 来源 id，歌曲时间 time，更新听歌排行数据

必选参数 : id: 歌曲 id, sourceid: 歌单或专辑 id

可选参数 : time: 歌曲播放时间,单位为秒

接口模块 : scrobble

黑胶乐签打卡
说明: 登录后调用此接口, 进行黑胶乐签打卡

接口模块: vip_sign

签到
说明 : 调用此接口 , 传入签到类型 ( 可不传 , 默认安卓端签到 ), 可签到 ( 需要登录 ), 其中安卓端签到可获得 3 点经验 , web/PC 端签到可获得 2 点经验

可选参数 : type: 签到类型 , 默认 0, 其中 0 为安卓端签到 ,1 为 web/PC 签到

接口模块 : daily_signin

云贝签到
说明 : 登录后调用此接口可进行云贝签到

接口地址 : yunbei_sign

四、页面设计

1. 根路径（"/"）：无静态页面，可根据 .env 中配置的 NETEASE_COOKIE 查看网易云音乐账号信息（如 Cookie 过期则提醒用户更换）。风格参考 ./参考/index.html。  
2. 登录路径（"/login"）：扫码登录页面，用户扫码登录网易云音乐账号后，自动返回账号的 Cookie 供用户复制（页面需包含“复制”按钮）。  参考./参考/login.html 和 ./参考/qrlogin.html
